# Marker 产品打磨 TODO

> 目标：按优秀产品标准，持续提升 `快记`、`低负担`、`可信`、`可回看`、`可扩展`。

## P0 必须优先

- [ ] 模板化创建追踪项，至少支持 `习惯`、`用药`、`经期`、`自定义`
- [ ] 重做 `Today` 首页结构，让用户一眼看懂 `今天该记什么 / 已经记了什么 / 还缺什么`
- [ ] 支持 `补记 / 改记 / 删除记录`
- [ ] 让 `History` 和 `Statistics` 真正理解不同 `payload`
- [ ] 做完整的本地可信能力：自动保存、撤销、导出/备份入口、明确本地状态

## P1 核心体验

- [ ] 追踪项支持 `类别 / 分组`
- [ ] `Today` 支持按类别筛选或折叠
- [ ] 不同追踪类型使用不同视觉样式和交互密度
- [ ] 历史页支持 `最近记录时间线`、`按天详情`、`类型筛选`
- [ ] 统计页按类型使用不同统计口径，而不是统一看成完成率
- [ ] 空状态、错误提示、删除确认、恢复反馈统一打磨
- [ ] 归档入口、低频配置、危险操作集中放在 `Settings`

## P1 记录模型

- [ ] 习惯型记录支持 `完成 + 备注`
- [ ] 用药型记录支持 `已服用 / 已跳过 / 剂量 / 单位 / 备注`
- [ ] 经期型记录支持 `流量 / 症状 / 备注`
- [ ] 自定义记录先支持 `备注型`，后续扩展为 `数值型 / 状态型`
- [ ] 历史明细直接展示 `payload` 摘要，而不是只显示“记录了一次”

## P1 信任感

- [ ] 清楚展示记录时间和逻辑日归属
- [ ] 清楚说明当前是 `本地优先`
- [ ] 清楚说明当前是否有 `同步 / 备份`
- [ ] 重要操作支持撤销或恢复
- [ ] 异常场景不能静默失败，要给出可理解反馈

## P2 高级增强

- [ ] 增加更通用的 `payload` 类型：`numeric / duration / range`
- [ ] 支持 `纯事件型 tracker`，不强依赖 `schedule`
- [ ] 支持 `导出`
- [ ] 支持 `备份 / 恢复`
- [ ] 支持 `小组件 / 快捷指令`
- [ ] 支持更智能的模板和默认配置

## 产品原则

- [ ] 不要把所有场景都按“习惯打卡”来设计
- [ ] 习惯场景强调坚持和反馈
- [ ] 用药场景强调准确、可信、低出错
- [ ] 经期场景强调隐私、细腻、可回看
- [ ] 自定义场景强调灵活和低门槛
