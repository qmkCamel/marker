## MODIFIED Requirements

### Requirement: 用户必须能够在 Today 中切换完成状态

系统必须（MUST）允许用户在 Today 视图中对某个追踪项创建、编辑或移除记录，并根据 `TrackerKind` 生成匹配的 payload。

对于简单完成型场景，系统可以使用快速记录；对于需要更多输入的类型，系统必须提供记录表单。记录表单必须（MUST）展示记录归属的逻辑日，并在删除已有记录前要求用户确认。

#### Scenario: 用户快速完成一个习惯型追踪项

- **WHEN** 用户在 Today 视图中将某个习惯型追踪项标记为完成
- **THEN** 系统为该追踪项记录当前逻辑日的一条完成型 payload 记录
- **AND** Today 视图立即反映新的完成状态和进度

#### Scenario: 用户记录一次用药事件

- **WHEN** 用户在 Today 视图中为用药型追踪项打开记录表单并保存
- **THEN** 系统为该追踪项记录当前逻辑日的一条用药型 payload 记录
- **AND** 记录中会保存该次用药所需的结构化字段

#### Scenario: 用户取消一个今日记录

- **WHEN** 用户在 Today 视图中移除某个已记录追踪项的今日记录
- **THEN** 当前逻辑日对应的追踪记录被移除
- **AND** Today 视图立即反映新的状态和进度

#### Scenario: 用户删除已有记录前看到确认

- **WHEN** 用户在记录表单中尝试删除已有记录
- **THEN** 系统展示删除确认
- **AND** 确认说明该删除会影响记录对应的逻辑日
- **AND** 用户确认后记录才会被删除
