# daily-tracking-workflow Specification

## Purpose
定义 Today 视图中的每日追踪工作流，包括应做列表、完成切换、空状态和进度反馈。
## Requirements
### Requirement: Today 视图必须展示当前逻辑日应完成的追踪项
系统必须（MUST）在 Today 视图中展示当前逻辑日应完成的活跃追踪项列表，并排除当日不应完成的追踪项与已归档追踪项。

#### Scenario: Today 只展示当前应做追踪项
- **WHEN** 用户进入 Today 视图
- **THEN** 视图展示当前逻辑日应完成的活跃追踪项
- **AND** 已归档追踪项与当天不应完成的追踪项不会出现在该列表中

### Requirement: 用户必须能够在 Today 中切换完成状态
系统必须（MUST）允许用户在 Today 视图中对某个追踪项创建或移除记录，并根据 `TrackerKind` 生成匹配的 payload。

对于简单完成型场景，系统可以使用快速记录；对于需要更多输入的类型，系统必须提供记录表单。

#### Scenario: 用户快速完成一个习惯型追踪项
- **WHEN** 用户在 Today 视图中将某个习惯型追踪项标记为完成
- **THEN** 系统为该追踪项记录当前逻辑日的一条完成型 payload 记录
- **AND** Today 视图立即反映新的完成状态和进度

#### Scenario: 用户记录一次用药事件
- **WHEN** 用户在 Today 视图中为用药型追踪项打开记录表单并保存
- **THEN** 系统为该追踪项记录当前逻辑日的一条用药型 payload 记录
- **AND** 记录中会保存该次用药所需的结构化字段

#### Scenario: 用户取消一个今日记录
- **WHEN** 用户在 Today 视图中移除某个已记录追踪项的今日记录
- **THEN** 当前逻辑日对应的追踪记录被移除
- **AND** Today 视图立即反映新的状态和进度

### Requirement: Today 视图必须提供清晰的空状态与进度反馈
系统必须（MUST）在没有活跃追踪项或当日无待办时，显示明确空状态，并在存在数据时提供当日进度反馈。

#### Scenario: 当前没有任何活跃追踪项
- **WHEN** 用户第一次进入应用且尚未创建任何追踪项
- **THEN** Today 视图显示引导性空状态
- **AND** 空状态提供创建追踪项入口
